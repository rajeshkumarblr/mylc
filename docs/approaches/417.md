The problem asks us to find all cells from which water can flow to both the Pacific Ocean (top and left edges) and the Atlantic Ocean (bottom and right edges). Water can flow from a cell to a neighboring cell if the neighbor's height is less than or equal to the current cell's height.

Instead of simulating water flowing down from every single cell (which would be very slow), we can think in reverse: start from the oceans and see where water can flow *up*.

A highly optimized approach uses a single state machine matrix instead of two separate boolean matrices:
- `0`: Unvisited
- `1`: Reachable from Pacific
- `2`: Reachable from Atlantic
- `3`: Reachable from both

1. **Pacific DFS:** First, we iterate along the top and left borders (Pacific edge) and perform a Depth-First Search (DFS). We only traverse to neighboring cells that are strictly $\ge$ the current cell's height. We mark these reachable cells as `1`.
2. **Atlantic DFS:** Next, we iterate along the bottom and right borders (Atlantic edge) and perform a second DFS. 
   - If we visit an unvisited cell (`0`), we mark it `2`.
   - If we visit a cell that was already marked `1` (by the Pacific DFS), we mark it `3` and immediately add it to our result array!
   - If a cell is already `2` or `3`, we can skip it to prevent infinite loops.
3. This elegant approach collects our answer *during* the Atlantic traversal, eliminating the need for a final full-matrix scan.

| Component | Complexity | Details |
|---|---|---|
| CPU | **O(M * N)** | where M and N are the dimensions of the grid. Each cell is processed by the Pacific DFS at most once, and by the Atlantic DFS at most once. |
| Memory | **O(M * N)** | for the state machine matrix `flow` (which can be optimized to use `char` for just 1 byte per cell) and the recursive call stack. |
